Programas con versión portable

Versión Instalable

Cuando el usuario ejecuta el instalador de un software, éste realiza automáticamente una serie de acciones sobre el sistema, como pueden ser:

  • Crear una carpeta en «archivos de programa» donde se copiarán los ficheros del programa.
  • Escribir nuevas claves en el registro de Windows o se modifican algunas de las existentes.
  • Crear una carpeta en %appdata% donde se guardarán las configuraciones de usuario.
  • Crear accesos directos en menú de inicio y escritorio.
  • Copiar librerías DLL necesarias en C:\Windows\System32

Como se puede ver, al instalar un programa se crean archivos en muchos lugares del sistema. Esto hace que, a la hora de desinstalar dicho programa, resulte difícil eliminar todo el rastro que ha dejado. Las versiones portables solucionan en gran medida este problema.

Versión portable

Cuando decimos que un software es portable, nos referimos a que para utilizarlo no es necesario pasar por un proceso de instalación.

Se le da el nombre de portable porque se puede copiar el directorio donde se encuentra almacenado y moverlo a otro ordenador sin que por ello deje de funcionar. Esto es posible ya que el ejecutable y todos los ficheros necesarios para su funcionamiento están dentro del mismo directorio, en lugar de estar desperdigados por el sistema operativo como ocurre con una versión instalable.

El software portable funciona de dos formas principales:

  • El ejecutable o su directorio contienen todas las librerías necesarias para su funcionamiento.
  • El software portable crea una máquina virtual donde se ejecutará mientras el usuario haga uso del programa. En cuanto el usuario cierra el programa la máquina virtual se destruye. Esta técnica es usada principalmente en software convertido a portable que originalmente necesitaba hacer uso del registro de Windows.

Entre las ventajas del software portable se encuentra el poder ser eliminado de un sistema sin dejar rastro, a diferencia de las versiones instalables, que suelen dejar ficheros y entradas en el registro. Esto depende mucho del programa, pero es cierto en la mayoría de los casos.

Una desventaja del software portable es que hay que configurar manualmente ciertas integraciones con el sistema operativo si se desea hacer uso de ellas como:

  • Accesos directos al menú de inicio.
  • Asociaciones de tipos de extensión (la primera vez que se abre ese tipo de fichero).
  • La adición de entradas al menú contextual a través de la configuración del programa.

En general es preferible usar un programa en su versión portable si es de pequeño tamaño y no nos importa crear manualmente las integraciones que necesitemos. Si se trata de software grande y que depende de muchas librerías externas, es preferible usar la versión instalable ya que su versión portable, al tener que almacenar duplicados de las librerías necesarias, puede llegar a ocupar mucho espacio.

En cuanto al funcionamiento del programa de cara al usuario (una vez configurado correctamente), se podría decir que no existe diferencia de uso y funcionalidad entre las versiones portables e instalables de un programa.

Versión portable (Linux)

En sistemas Linux es menos común el uso de versiones portables. Esto no quiere decir que no existan formas de distribuirlas. Se han probado dos: la que usa Blender y la que usa Krita.

  • En el caso de Blender se utiliza un método similar al de Windows. Se empaqueta el programa en un fichero comprimido para que el usuario lo descomprima donde desee. Una vez descomprimido se puede ejecutar sin problema.
  • Krita utiliza una forma de distribución llamada «AppImage» en el que todo el programa se empaqueta en un único fichero. Una vez descargado, le damos permiso de ejecución mediante el comando chmod y estaría listo para utilizarlo haciendo doble clic sobre él.
